# ZK 투표 시작하기

ZK-Vote는 **Zero-Knowledge Proof(영지식 증명)**를 사용해 **누가 무엇에 투표했는지 드러내지 않은 채** 투표할 수 있는 서비스입니다.

## ZK 투표란?

- **익명성(Anonymity)**: 투표 내용과 본인을 연결해 보여 주지 않습니다.
- **검증 가능성(Verifiability)**: "등록된 투표자만 투표했다", "한 사람 한 표"는 블록체인에서 검증 가능합니다.
- **이중 투표 방지**: 한 번만 투표할 수 있고, 중복 투표는 nullifier로 차단됩니다.

즉, **투표했다는 사실만 증명**하고, **어떤 선택을 했는지·누구인지는 공개하지 않는** 방식입니다.

## 관리자 vs 사용자(투표자)

역할에 따라 하는 일이 다릅니다.

| 역할               | 설명                                                                                                                        |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| **관리자(Admin)**  | 투표를 **생성**하고, 투표자를 **등록**하며, **등록 마감**을 하는 쪽입니다. 관리자 웹에서 Google 로그인 후 진행합니다.       |
| **사용자(투표자)** | 투표에 **참여 등록**한 뒤, **ZK proof를 생성**하고 **투표**를 제출하는 쪽입니다. 사용자 웹에서 Google 로그인 후 진행합니다. |

## 전체 흐름 요약

1. **관리자: 투표 등록** — 투표 제목, 설명, 후보(선택지), 기간을 정해 투표 세션을 만듭니다.
2. **관리자: 투표자 등록** — 참여할 사람들이 사용자 앱에서 등록 요청을 보내면, 관리자가 그 commitment를 컨트랙트에 등록합니다(또는 시스템이 자동 반영).
3. **관리자: 투표 등록 마감** — 더 이상 투표자를 받지 않고, Merkle root를 설정한 뒤 투표 기간을 시작합니다.
4. **사용자: 증명(proof) 생성** — 투표 시, 브라우저에서 본인의 secret으로 ZK proof를 생성합니다.
5. **사용자: 투표** — 생성한 proof를 담아 `castVote` 트랜잭션을 제출합니다. 한 번만 투표 가능합니다.

자세한 단계는 [전체 흐름](/docs/zk-vote/getting-started/quick-start)에서 확인할 수 있습니다.
