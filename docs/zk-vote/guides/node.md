# 노드에 대해

ZK 투표는 **블록체인** 위에서 동작합니다. 그 블록체인을 구성하는 **노드**에 ZK 증명을 검증하는 기능이 들어 있어야, “투표한 사람이 등록된 참여자다”, “한 번만 투표했다” 같은 검증이 온체인에서 이루어집니다.

엔드 유저 입장에서는 **노드를 직접 설치·운영할 필요는 없습니다.** 다만 “내 투표가 어떤 환경에서 검증되는지” 정도만 알아두면 됩니다.

## 노드가 하는 일

- **일반 블록체인 노드**: 트랜잭션 전파, 블록 생성·검증, 상태 저장
- **ZK 관련 추가 기능**: 스마트 컨트랙트가 **Groth16 ZK 증명 검증**을 요청하면, 노드에 내장된 **프리컴파일(Precompile)** 이 그 증명을 검증합니다. 검증에 성공해야 투표 트랜잭션이 유효하게 처리됩니다.

즉, “투표 트랜잭션 안에 들어 있는 ZK 증명이 진짜인지”를 **노드가** 검증해 주고, 그 결과에 따라 컨트랙트가 투표를 집계합니다.

## 사용자가 신경 쓸 것

- **네트워크 선택**: ZK 투표 앱에서 연결하는 네트워크(체인)가 “ZK 검증을 지원하는 노드”들이 붙어 있는 네트워크여야 합니다. 서비스 제공자가 안내한 **RPC 주소**나 **네트워크 이름**을 쓰면 됩니다.
- **노드 직접 운영**: 일반 사용자는 노드를 돌리지 않아도 됩니다. 서비스·운영자가 노드를 구성해 두고, 사용자는 **앱에 로그인**해 투표만 하면 됩니다.

## 정리

- ZK 투표의 **메인**은 **앱에서 등록·투표·결과 확인**하는 것입니다.
- **노드**는 그 투표 트랜잭션과 ZK 증명을 검증해 주는 “블록체인 쪽” 구성 요소이며, 엔드 유저는 “ZK를 지원하는 네트워크에 연결된 앱을 쓴다” 정도만 알면 충분합니다.

노드 구축·설정이 궁금한 경우에는 서비스 운영자 또는 개발자 문서를 참고하면 됩니다.
